Resumen de las lecturas sugeridas

Los casos de uso

Estos tambien son diagramas que se emplean para estructurar las actividades que va realizar una aplicacion que se esta diseñando, con respecto a los deseos y objetivos que tiene el cliente. Este facilita el entendiemiento de todos los roles, junto con las actividades en las que cada uno de ellos va a participar.

Estos describen las acciones,interacciones, comportamientos, límites del sistema y su relación con el entorno, todo visto desde el punto de vista de los actores que con él negocian.

Los diagramas de uso cuentan con unas partes, las cuales se deben tener en cuenta al momento de realizar el mismo.

1- Actores
2- El caso se uso
3- Las relaciones
4- Asociacion
5- Inclusion
6- Extension
7- Generalización y especialización
8- Documentación del caso de uso

Ingenieria de requerimientos

Se conoce como uno de los procesos mas importantes dentro de la recoleccion de la informacion con respecto al aplicativo que se desea crear. Este proceso es recientemente conocido y se ha iniciado a implementar para hacer el levantamiento de los requisitos necesarios para el funcionamiento y el buen desarrollo de una producto que se desea fabricar. Al momento de solicitar u obtener los requerimientos se deben tener en cuenta tres puntos importantes, los cuales son:

1- Necesidades
2- Funcionalidades
3- Requerimientos del software.

Modelado de los requisitos

Este método se emplea para hace un vinculo, entre los roles que tienen cada una de las personas que hacen parte de la cempresa ya sea de manera directa e indirecta. Este facilita la distribucion de las actividades que van a realizar cada uno de los individuos que hagan diferentes actividades en la aplicacion.

Es importante tener en cuenta que todos los actores se representan de la misma manera. Una de las alternativas al emplear UML, es que podemo agregar cardinalidad a lo que se esta desarrollando, teniendo en cuenta las cantidades de casos que se deben ejecutar por cada actor.

Es necesario saber que los valores que se emplean para estos caso varian. Basados en la siguiente tabla.

Especificacion Cardinalidad
0..1 Cero o una vez
1 Una única vez
* De cero a varias veces
1..* De una a varias veces
M..N Entre M y N
N N veces

La inclusion dentro de los casos de usos sirven para compartir una funcionalidad en común en varios casos. Y tambien se pueden emplear las extensiones para agregarle subfunciones a los casos de inclusion.

Los casos de uso nos brindan una variedad de tecnicas que nos permiten hacer una representacion idonea para tener un dialogo con el usuario ya que este se maneja con un lenguaje natural. Todo esto llegando facilitar el entendimiento de lo que se desea y se planifica desarrollar.
